## 🚀 Usage

### In Controllers

```php
// Load translation
use RunwayHub\Core\Translator;

$translator = app(Translator::class);
echo $translator->get('users::login_button');
```

### In Views

```php
// Helper function
echo __('users::login_button');

// Or with context
echo __('users::welcome', ['name' => $user->name]);
```

### CLI Commands

```bash
# Extract strings
php artisan i18n:extract

# Update POT files
php artisan i18n:translate

# Validate translations
php artisan i18n:validate
```

## 🔧 Implementation Guide

### Adding New Language

1. **Create translation directory**
   ```bash
   mkdir -p i18n/fr
   ```

2. **Copy base translations**
   ```bash
   cp i18n/en/*.php i18n/fr/
   ```

3. **Translate strings**
   ```bash
   # Use POEdit or similar
   pootle/
   ```

4. **Test translations**
   ```bash
   php artisan i18n:test --lang=fr
   ```

### Best Practices

1. **Keep translations consistent** - Same terminology
2. **Context matters** - Pass context to translators
3. **Gender/number** - Handle German gender/number correctly
4. **RTL support** - Plan for Arabic/Hebrew if needed
5. **Fallback chain** - en → de → fallback

## 🌐 Localization vs Translation

### Localization (l10n)

- Date formats
- Number formatting
- Currency symbols
- Time zones

```php
// Example
echo date('d.m.Y', strtotime($now)); // German format
echo number_format(1234.56, 2, ',', '.'); // German numbers
echo \Illuminate\Support\Facades\App::currency(978); // EUR
```

### Translation (i18n)

- Text content
- UI labels
- Error messages
- Email templates

## 📝 Common Translation Issues

### Issue 1: Missing Translation

**Solution:**
```php
// Use fallback
echo __('missing::key') ?? __('fallback::key');
```

### Issue 2: Hardcoded Strings

**Fix:**
```php
// Before
echo 'Hello World';

// After
echo __('users::welcome');
```

### Issue 3: Context Loss

**Solution:**
```php
// Good - passes context
__('errors::validation', [
    'field' => $field,
    'value' => $value,
]);
```
