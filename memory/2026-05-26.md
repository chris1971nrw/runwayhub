## 📝 Key Decisions Today

1. **Cache Strategy:** 5-min TTL for OpenAIP/Weather, 2-min for FlightAware
2. **Rate Limiting:** 2s backoff on 429 errors
3. **Documentation:** Document as you code, don't wait
4. **Security:** Environment variables for all secrets
5. **Blog Content:** Tutorial-first approach

### Rationale

**Why 5-min cache:**
- Aviation data changes slowly
- Reduces API calls significantly
- Still provides fresh data
- Balance between accuracy and performance

**Why 2-min for FlightAware:**
- Live flight positions change faster
- Users expect near real-time
- Still reduces API load

**Why 2s backoff:**
- Avoids hammering API
- Gives server time to recover
- Prevents cascading failures
